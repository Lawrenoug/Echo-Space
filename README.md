# Echo Space

`Echo Space` 是一个使用 `Godot 4.6.2 + C#` 开发的 2D 横版动作游戏原型，核心方向是：

- 现实世界 / 灵魂世界实时切换
- 长横向、多层结构地图探索
- 以《只狼》为参考方向的近战战斗
- 逐步搭建可扩展的系统框架，而不是一次性堆完整内容

## 当前战斗方向

当前战斗设计方向已经明确为“血量 + 耐力 + 架势 + 处决”的组合：

- 玩家拥有血量和耐力
- 敌人拥有血量和架势
- 普通攻击会削减敌人血量，并附带较少的架势伤害
- 防御可以减轻压力，精准弹反会更快累积敌人架势
- 当敌人架势打满后，会进入可处决状态
- 玩家贴近破绽敌人时，可通过一次处决攻击直接终结敌人

## 当前默认按键

- `A` / `Left`：向左移动
- `D` / `Right`：向右移动
- `Space` / `W` / `Up`：跳跃
- `鼠标左键`：攻击 / 处决
- `鼠标右键`：防御 / 弹反
- `Tab`：切换世界
- `I`：打开 / 关闭背包界面
- `P`：打开 / 关闭加点界面
- `Esc`：关闭当前最上层系统界面
- `F5`：快速存档
- `F9`：快速读档

## 当前框架说明

- 项目启动入口为主菜单场景，菜单预留了开始游戏、继续游戏、设置和退出接口
- 继续游戏已经接入基础存档系统，可读取最近一次快速存档
- 存档系统当前采用单文件方案，保存玩家位置、玩家血量与耐力、当前世界、背包内容、加点数据、双世界物体状态
- 双世界物体通过 `save_id` 参与存档，当前原型中的双世界平台和可破坏墙都已接入
- 二级系统界面当前采用统一的“面板栈”逻辑，切换界面时下层界面保留，关闭顶层后自动恢复
- 背包系统当前负责：物品定义、分类、槽位、堆叠、增删查、变化事件、基础存档序列化
- 加点系统当前负责：等级、未分配点数、属性分配 / 退点 / 重置、战斗修正值快照、基础存档序列化
- 存档系统当前以“先搭骨架、后扩展为正式存档槽位界面”为目标，后续适合在现有结构上继续扩展
